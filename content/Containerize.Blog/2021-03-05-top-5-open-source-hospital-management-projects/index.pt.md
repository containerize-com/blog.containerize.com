## openemr   {#opeNemr}
[OpenEMR][10] é um sistema de informações hospitalares de código aberto para gerenciar registros de saúde. É multilíngue e oferece muitos recursos, como prescrição eletrônica, gerenciamento de cobrança e muito mais. Além disso, essa solução de TI de saúde é inteligente o suficiente para fornecer unidades de saúde remotas. Este  **Sistema de Gerenciamento Hospitalar **  oferece criptografia de dados para garantir a segurança dos dados e criar um menu baseado em função que seja totalmente personalizável. Omeremr é multilíngue, extensível e fornece um sistema de cobrança integrado. Além disso, ele fornece uma interface amigável, onde os usuários podem gerar relatórios personalizados. O OpenEMR está escrito no PHP e vem com toda a documentação sobre desenvolvimento e desenvolvimento. Portanto, todo o código -fonte está disponível no [Github][11].
OpenEMR oferece os seguintes pontos -chave:
  * Agendar compromissos dos pacientes
  * Criptografia de dados
  * Prescrição eletrônica
  * Plataforma cruzada
  * Integração do laboratório
[Saiba mais sobre o OpenEMR][12]

## OpenMrs   {#openMrs}
[OpenMrs][13] é um software de gerenciamento de registros de pacientes adaptáveis ​​de código aberto. Essa solução de saúde baseada na Web fornece módulos poderosos, como fluxos de trabalho de pacientes incorporados, login baseado em localização e vários identificadores. O OpenMRS é construído sobre a arquitetura modular, é multilíngue e oferece suporte para muitos idiomas. Além disso, este sistema de gerenciamento de registros de saúde tem uma comunidade forte que tem trabalhado constantemente para desenvolver outros módulos e plugins. Acima de tudo, este sistema de saúde eletrônico de código aberto  **oferece uma interface RESTful para aplicativos de terceiros. Este **  software médico de código aberto ** está escrito em Java e toda a documentação está disponível. Portanto, seu código -fonte está disponível no [Github][14].
OpenMrs cria os seguintes recursos principais:
  * Módulo de registro do paciente
  * Seguro
  * Arquitetura modular
  * Permissões baseadas em funções
  * Interface do dicionário
[Saiba mais sobre o OpenMRS][15]

## Hospitalrun   {#hospitalrun}
[HospitalRun][16] é outro sistema de informações médicas premiadas e de código aberto. Ele fornece versões on -line e offline para a facilidade dos usuários. Acima de tudo, existe um mecanismo de sincronização para manter os dados sincronizados nas plataformas integradas. O HospitalRun oferece módulos para lidar com casos de emergência e o sistema de alta do paciente. Além disso, existe um poderoso recurso de pesquisa neste projeto de gerenciamento hospitalar de código aberto. Ele oferece uma interface de usuário intuitiva para usuários e um sistema de cobrança integrado. Além disso, este sistema de informação do paciente está escrito em JavaScript e toda a documentação sobre desenvolvimento e implantação está disponível. Portanto, seu código -fonte está disponível no [Github][17].
Hospitalrun oferece os seguintes recursos importantes:
  * Auto-hospedado
  * Sistema de cobrança personalizado
  * Agendamento de compromissos
  * Registros médicos eletrônicos
  * Gerenciamento de usuários
[Saiba mais sobre Hospitalrun][18]

Hospital Open#{#Open-Hospital}
[Hospital Open][19] é um sistema de gerenciamento hospitalar gratuito  **com vários recursos para automatizar hospitalizações, visitas aos pacientes, dados de laboratório, histórico de tratamento e outros recursos de integração suave. Além disso, é fácil configurar os servidores locais/em nuvem e oferece API REST para integrar-se a outros aplicativos de terceiros. Além disso, existe um módulo que permite aos usuários gerenciar dados de laboratório neste projeto de gerenciamento hospitalar. Este **  software médico de código aberto ** é multilíngue e pode ser modificado conforme as funcionalidades necessárias. O Hospital Open permite que os funcionários da saúde TP automatizem a submissão e as visitas ao paciente. Além disso, o Open Hospital está escrito em Java e a documentação abrangente sobre o desenvolvimento está disponível. Portanto, você pode encontrar seu código -fonte no [Github][20].
O Hospital Open tem os seguintes recursos importantes:
  * Multilíngue
  * Banco de dados de vacinas
  * Submissão do paciente
  * API REST
  * Gerenciamento de cobrança
[Saiba mais sobre o Hospital Aberto][21]

## Solismed   {#solismed}
[Solismed][22] é um sistema ambulatorial e de gerenciamento ambulatorial de código aberto para a indústria médica. Este  **sistema de saúde eletrônico **  é seguro, escalável e oferece muitos recursos ricos, como integração de laboratório, lembretes de consultas e um pouco mais. Além disso, permite que seus usuários configurem as tarefas de rotina apenas na interface do usuário. Solismed surge com recursos de auto-hospedagem. Este  **Software médico de código aberto **  fornece gerenciamento de medicamentos e serviços de cobrança robustos. Está escrito no PHP e toda a documentação está disponível. Portanto, seu código -fonte está disponível neste [link][23], juntamente com todos os detalhes de desenvolvimento e implantação.
A seguir estão os recursos proeminentes:
  * Demografia de pacientes
  * Serviço de cobrança médica
  * Portal do paciente
  * Controle de estoque de drogas
  * Agendamento de compromissos
[Saiba mais sobre solismo][24]
