## OpenEMR {#OpenEMR}
[OpenEMR] [10] est un système d'information hospitalier open source pour gérer les dossiers de santé. Il est multilingue et offre de nombreuses fonctionnalités telles que la prescription électronique, la gestion de facturation et bien d'autres. De plus, cette solution informatique de soins de santé est suffisamment intelligente pour fournir des établissements de santé à distance. Ce ** Système de gestion des hôpitaux ** propose un chiffrement des données pour s'assurer que la sécurité des données et propose un menu basé sur les rôles qui est entièrement personnalisable. Omeremr est multilingue, extensible et fournit un système de facturation intégré. De plus, il fournit une interface conviviale où les utilisateurs peuvent générer des rapports personnalisés. OpenEMR est écrit en PHP et est livré avec toutes les documents concernant le développement et le développement. Par conséquent, tout le code source est disponible chez [GitHub] [11].
OpenEMR propose les points clés suivants:
  * Planifiez les rendez-vous des patients
  * Encryption de données
  * prescription électronique
  * Multiplateforme
  * Intégration de laboratoire
[En savoir plus sur OpenEMR] [12]

## OpenMrs {#OpenMrs}
[OpenMRS] [13] est un logiciel de gestion des enregistrements de patients adaptable open source. Cette solution de soins de santé sur le Web fournit des modules puissants tels que les flux de travail des patients intégrés, la connexion basée sur la localisation et plusieurs identifiants. OpenMRS est construit sur l'architecture modulaire, est multilingue et offre une prise en charge de nombreuses langues. De plus, ce système de gestion des dossiers de santé a une communauté solide qui a toujours travaillé pour développer d'autres modules et plugins. Surtout, ce système de santé électronique open source ** offre une interface reposante pour les applications tierces. Ce ** logiciel médical open source ** est écrit en Java et toute la documentation est disponible. Par conséquent, son code source est disponible chez [github] [14].
OpenMRS propose les caractéristiques clés suivantes:
  * Module d'enregistrement du patient
  * Sécurisé
  * Architecture modulaire
  * Autorisations basées sur les rôles
  * Interface du dictionnaire
[En savoir plus sur les OpenMR] [15]

## Hospitalrun {#hospitalrun}
[Hospitalrun] [16] est un autre système d'information médicale primée gratuit et open source. Il fournit des versions en ligne et hors ligne pour la facilité des utilisateurs. Surtout, il existe un mécanisme de synchronisation pour maintenir les données synchronisées sur les plates-formes intégrées. Hospitalrun propose des modules pour traiter les cas d'urgence et le système de sortie du patient. De plus, il existe une puissante fonctionnalité de recherche dans ce projet de gestion hospitalière open source. Il offre une interface utilisateur intuitive pour les utilisateurs et un système de facturation intégré. De plus, ce système d'information sur le patient est rédigé en JavaScript et toute la documentation concernant le développement et le déploiement est disponible. Par conséquent, son code source est disponible chez [GitHub] [17].
HospitalRun propose les caractéristiques importantes suivantes:
  * Hostomé
  * Système de facturation personnalisé
  * Planification de rendez-vous
  * Dossiers médicaux électroniques
  * Gestion des utilisateurs
[En savoir plus sur l'hôpital] [18]

## Open Hospital {# Open-Hospital}
[Open Hospital] [19] est un système de gestion hospitalier ** gratuit ** avec plusieurs caractéristiques pour automatiser les hospitalisations, les visites aux patients, les données de laboratoire, les antécédents de traitement et d'autres caractéristiques d'intégration en douceur. De plus, il est facile de configurer sur les serveurs locaux / cloud et propose une API REST pour s'intégrer à d'autres applications tierces. De plus, il existe un module qui permet aux utilisateurs de gérer les données de laboratoire dans ce projet de gestion hospitalière. Ce ** logiciel médical open source ** est multilingue et peut être modifié selon les fonctionnalités nécessaires. L'hôpital ouvert permet aux responsables de la santé TP automatiser la soumission des patients et les visites des patients. De plus, l'Open Hospital est rédigé en Java et une documentation complète concernant le développement est disponible. Par conséquent, vous pouvez trouver son code source à [GitHub] [20].
L'hôpital ouvert a les caractéristiques clés suivantes:
  * Multilingue
  * Base de données vaccinales
  * Soumission du patient
  * API REST
  * Gestion de facturation
[En savoir plus sur l'Open Hospital] [21]

## Solismed {#solismed}
[Solismemed] [22] est un système de gestion ambulatoire et hospitalier open source pour l'industrie médicale. Ce ** Système de santé électronique ** est sécurisé, évolutif et offre de nombreuses fonctionnalités riches telles que l'intégration de laboratoire, les rappels de rendez-vous et d'autres. De plus, il permet à ses utilisateurs de configurer les tâches de routine uniquement à partir de l'interface utilisateur. Solismed propose des capacités d'auto-hébergement. Ce ** Open Source Medical Software ** fournit des services de gestion de médicaments et de facturation robuste. Il est écrit en PHP et toutes les documents sont disponibles. Par conséquent, son code source est disponible dans ce [lien] [23] ainsi que tous les détails de développement et de déploiement.
Voici les caractéristiques importantes:
  * Démographie du patient
  * Service de facturation médicale
  * Portail des patients
  * Contrôle de la drogue
  * Planification de rendez-vous
[En savoir plus sur le solismed] [24]
