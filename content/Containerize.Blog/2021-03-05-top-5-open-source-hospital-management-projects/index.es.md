## Openemr {#openemr}
[OpenEmr][10] es un sistema de información hospitalaria de código abierto para gestionar los registros de salud. Es multilingüe y ofrece muchas características, como prescripción electrónica, gestión de facturación y muchas más. Además, esta solución de TI de atención médica es lo suficientemente inteligente como para proporcionar instalaciones de salud remotas. Este ** Sistema de gestión del hospital ** ofrece cifrado de datos para asegurar la seguridad de los datos y se le ocurre un menú basado en roles que es totalmente personalizable. Omeremr es multilingüe, extensible y proporciona un sistema de facturación integrado. Además, proporciona una interfaz fácil de usar donde los usuarios pueden generar informes personalizados. OpenEMR está escrito en PHP y viene con toda la documentación con respecto al desarrollo y el desarrollo. Por lo tanto, todo el código fuente está disponible en [GitHub][11].
OpenEMR ofrece los siguientes puntos clave:
  * Programe las citas de los pacientes
  * Cifrado de datos
  * prescripción electrónica
  * Multiplataforma
  * Integración de laboratorio

## OpenMrs {#openmrs}
[OpenMRS][13] es un software de gestión de registros de pacientes de código abierto de código abierto. Esta solución de salud basada en la web proporciona módulos potentes, como flujos de trabajo de pacientes integrados, inicio de sesión basados ​​en la ubicación e identificadores múltiples. OpenMRS está construido sobre la arquitectura modular, es multilingüe y ofrece soporte para muchos idiomas. Además, este sistema de gestión de registros de salud tiene una comunidad sólida que ha estado trabajando constantemente para desarrollar más módulos y complementos. Sobre todo, este sistema de salud electrónico ** de código abierto ** ofrece una interfaz RESTful para aplicaciones de terceros. Este ** software médico de código abierto ** está escrito en Java y toda la documentación está disponible. Por lo tanto, su código fuente está disponible en [Github][14].
OpenMRS presenta las siguientes características clave:
  * Módulo de registro del paciente
  * Seguro
  * Arquitectura modular
  * Permisos basados ​​en roles
  * Interfaz del diccionario

## Hospitalrun {#Hospitalrun}
[HospitalRun][16] es otro sistema de información médica galardonada con código abierto y gratuito. Proporciona versiones en línea y fuera de línea para la facilidad de los usuarios. Sobre todo, existe un mecanismo de sincronización para mantener los datos sincronizados en las plataformas integradas. HospitalRun ofrece módulos para tratar los casos de emergencia y el sistema de alta del paciente. Además, hay una poderosa característica de búsqueda en este proyecto de gestión de hospitales de código abierto. Ofrece una interfaz de usuario intuitiva para los usuarios y un sistema de facturación integrado. Además, este sistema de información del paciente está escrito en JavaScript y toda la documentación con respecto al desarrollo y la implementación está disponible. Por lo tanto, su código fuente está disponible en [GitHub][17].
HospitalRun ofrece las siguientes características importantes:
  * Autohostado
  * Sistema de facturación personalizado
  * Programación de citas
  * Registros médicos electrónicos
  * Gestión de usuarios
[Obtenga más información sobre HospitalRun][18]

## Open Hospital {#abre-hospital}
[Open Hospital][19] es un sistema gratuito de gestión del hospital ** con múltiples características para automatizar hospitalizaciones, visitas al paciente, datos de laboratorio, historial de tratamiento y otras características de integración suaves. Además, es fácil de configurar en servidores locales/en la nube y ofrece API REST para integrarse con otras aplicaciones de terceros. Además, hay un módulo que permite a los usuarios administrar datos de laboratorio en este proyecto de gestión del hospital. Este ** software médico de código abierto ** es multilingüe y puede modificarse según las funcionalidades necesarias. Open Hospital permite a los funcionarios de atención médica TP automatizar el presentación del paciente y las visitas al paciente. Además, Open Hospital está escrito en Java y está disponible una documentación integral con respecto al desarrollo. Por lo tanto, puede encontrar su código fuente en [GitHub][20].
Open Hospital tiene las siguientes características clave:
  * Multilingüe
  * Base de datos de vacunas
  * Presentación del paciente
  * API REST
  * Gestión de facturación
[Obtenga más información sobre Open Hospital][21]

## Solismed {#SolisMed}
[Solismado][22] es un sistema de gestión de pacientes ambulatorios y pacientes hospitalizados para la industria médica. Este ** sistema de salud electrónica ** es seguro, escalable y ofrece muchas características ricas como integración de laboratorio, recordatorios de citas y algunos más. Además, permite a sus usuarios configurar las tareas de rutina solo desde la interfaz de usuario. Solismed se le ocurre las capacidades de autohospedación. Este ** Software médico de código abierto ** proporciona servicios de facturación sólidos y de gestión de medicamentos. Está escrito en PHP y toda la documentación está disponible. Por lo tanto, su código fuente está disponible en este [enlace][23] junto con todos los detalles de desarrollo e implementación.
Las siguientes son las características destacadas:
  * Datos demográficos del paciente
  * Servicio de facturación médica
  * Portal de pacientes
  * Control de stock de drogas
  * Programación de citas
