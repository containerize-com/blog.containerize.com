## автоматически отслеживает ваши веб -сайты, веб -приложения, API с бесплатными инструментами мониторинга и демонстрации времени Upcame Services с помощью системы статуса с открытым исходным кодом.

{{< figure align=center src="images/How-to-monitor-websites-with-statping.png" alt="Мониторинг веб -приложений">}}

Законевание - это программное обеспечение для страницы статуса с открытым исходным кодом ** и один из лучших инструментов мониторинга веб -сайтов **. Он также использует для мониторинга услуг, таких как HTTP, TCP, UDP, ICMP и GRPC. Готово автоматически получает услуги, контролирует их и делает красивую страницу статуса. Он кроссплатформенный, и вы можете установить его на операционные системы LiUnx, Mac и Windows. Кроме того, он позволяет использовать систему баз данных MySQL, Postgres или SQLite для хранения информации. Более того, он поставляется с мобильными приложениями Box для устройств iPhone и Android.
Давайте посмотрим на основные функции ставки.
  *** Мониторинг сервисов ** - Законевание позволяет легко отслеживать все ваши услуги HTTP, TCP, UDP, GRPC и ICMP.
  *** Уведомления **-Законевание поддерживает большинство основных сторонних приложений для получения уведомлений.
  *** Мобильное приложение ** - Используйте бесплатное приложение для устройств для iPhone и Android и управляйте всем на ходу.
  *** OAuth Authenticatio ** n - поддерживает аутентификацию OAuth для популярных услуг, таких как Github, Google, Slack и Custom OpenID -поставщик. Используйте эту функцию и повысите безопасность.


## configure systemd service {#configure}
Настройка службы Systemd - отличный способ обеспечить, чтобы ваш сервер законеков мог автоматически перезагружаться при необходимости. Чтобы создать системную службу, выполните следующие шаги.
  * Сначала запустите команду ниже, чтобы создать новый файл.
```
sudo nano /etc/systemd/system/statping.service
```
  * Скопируйте ниже код и вставьте в него.
```
[Unit]
Description=Statping Server
After=network.target
After=systemd-user-sessions.service
After=network-online.target

[Service]
Type=simple
Restart=always
ExecStart=/usr/local/bin/statping
WorkingDirectory=/usr/local/bin

[Install]
WantedBy=multi-user.target
```
  * Наконец, используйте ниже команды, чтобы включить и запустить Systemd Service.
```
systemctl daemon-reload
systemctl enable statping
systemctl start statping
```

## Создать службу для мониторинга {#Create}
  * Откройте свой браузер, введите URL для входа в систему для установки законодательных ставок.
  * Нажмите на ссылку Services в Top Navigation. Он покажет вам страницу услуг, которая выглядит как ниже.

{{< figure align=center src="images/statping-dashboard.png" alt="Список мониторинга веб -сервисов">}}

  * Нажмите кнопку «Создать», чтобы добавить новую службу для мониторинга. Заполните требуемые поля, такие как имя обслуживания, тип обслуживания, интервал проверки, конечная точка службы (URL) и т. Д.
